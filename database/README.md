# SMS Database Update Folder

This folder contains database documentation and draft SQL for the updated frontend and backend plan.

The SQL is a scaffold for the next database phase. It should be reviewed against the existing database before applying so we do not duplicate existing tables.

## Covered Areas

- Schools and tenant settings
- Roles and permissions
- Admissions and admission number sequences
- Secretary/front desk imports
- Bursar finance imports
- Staff management
- Student accounts and cashless payments
- Receipts, invoices, requisitions, payment vouchers
- Finance documents and share-to-admin approvals
- Budgets, vote book, cashbooks, bank reconciliation
- Library books, copies, shelves, borrowing, requested books
- Communication by SMS, email, push, and in-app notifications
- Parent-teacher messaging
- Audit logs and red flags
- Export/report jobs

## Security Notes

- Do not store plaintext passwords.
- Password hashes should be generated by the backend.
- All tenant data must be scoped by `school_id`.
- Sensitive actions should write to `audit_logs`.
- Large reports should use `export_jobs`.
